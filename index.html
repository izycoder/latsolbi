5.1 Definisi & Konsep Dasar Data Mining

Data Mining (Fayyad et al., 1996): Proses nontrivial dalam mengekstraksi pengetahuan yang valid (dapat dipercaya), novel (baru), potentially useful (potensial bermanfaat), dan understandable (dapat dipahami) dari data dalam jumlah besar.
🔑 Penjelasan 5 Kata Kunci + Nama Lain
+
NONTRIVIAL

Bukan perhitungan sederhana. Membutuhkan algoritma kompleks, tidak bisa dilakukan manual. Contoh: menemukan pola dari jutaan transaksi.

VALID

Pola harus akurat dan dapat diandalkan — bukan kebetulan statistik. Harus divalidasi dengan test set (data yang belum pernah dilihat model).

NOVEL

Pengetahuan yang dihasilkan harus baru — sesuatu yang belum diketahui sebelumnya. Jika sudah diketahui, tidak ada nilai tambahnya.

POTENTIALLY USEFUL

Harus bisa menghasilkan manfaat nyata: keputusan bisnis lebih baik, penghematan biaya, peningkatan revenue.

UNDERSTANDABLE

Hasil harus bisa dipahami oleh manusia (terutama pengambil keputusan). Ini mengapa interpretability adalah kriteria evaluasi penting.

Nama lain Data Mining:

Knowledge ExtractionPattern AnalysisKnowledge Discovery in Databases (KDD)Information HarvestingData DredgingPattern Searching
Mengapa "misnomer"? Yang "digali" BUKAN datanya — data sudah tersedia. Yang digali adalah knowledge / pola tersembunyi di dalam data.

5 Faktor Pendorong Data Mining:

  1. Pengakuan nilai dari sumber data yang tersedia
  2. Ketersediaan data berkualitas (pelanggan, vendor, transaksi, web)
  3. Konsolidasi data ke dalam data warehouse
  4. Peningkatan eksponensial kemampuan komputasi + penurunan biaya
  5. Konversi informasi ke bentuk digital/non-fisik

Environment DM: Biasanya client-server atau web-based. Data = bahan paling kritis. Miner seringkali end user langsung. "Striking it rich requires creative thinking."

5.2 Tiga Proses Standar Data Mining

🔄 CRISP-DM — 6 Langkah (Paling Populer)
+

Proses ini sangat iteratif — bisa kembali ke langkah sebelumnya kapan saja. DM = art vs science.

1
Business Understanding
Pahami tujuan bisnis
Apa: Tentukan tujuan bisnis dari perspektif bisnis (bukan teknis).

Contoh: "Kurangi churn pelanggan 20% dalam 6 bulan."

Output: Business objectives, success criteria, project plan
2
Data Understanding
Kenali data
Apa: Kumpulkan data awal, kenali karakteristik, identifikasi masalah kualitas data.

Contoh: Cek missing values, distribusi, outlier.

Output: Data description report
3
Data Preparation
Bersihkan & siapkan
Apa: Seleksi, pembersihan, konstruksi, integrasi data. Paling memakan waktu (60-80% total waktu proyek)!

Contoh: Imputasi missing values, encoding, normalisasi.
4
Modeling
Pilih & terapkan algoritma
Apa: Pilih teknik, buat test design, bangun dan nilai model.

Contoh: Coba DT, SVM, ANN, bandingkan hasilnya.
5
Evaluation
Nilai model vs tujuan bisnis
Apa: Evaluasi apakah model memenuhi tujuan BISNIS (bukan hanya akurasi teknis).

Contoh: Model 90% akurat tapi pakai variabel yang tidak bisa dikumpulkan di produksi → tidak layak deploy.
6
Deployment
Terapkan ke produksi
Apa: Integrasikan model ke dalam proses bisnis.

Contoh: Model rekomendasi diintegrasikan ke website e-commerce.
INGAT: Proses bersifat iteratif. Bisa kembali ke langkah manapun kapan saja. Ini bukan proses satu arah!
📊 SEMMA (SAS Institute) — 5 Langkah
+
S — Sample

Ambil sampel representatif dari data besar. Cukup besar untuk mengandung pola, cukup kecil untuk diproses efisien.

E — Explore

Eksplorasi data (EDA): visualisasi, summary statistics, deteksi outlier, identifikasi hubungan antar variabel.

M — Modify

Modifikasi dan transformasi data: buat variabel baru, ubah skala, handle missing values, encode kategoris.

M — Model

Terapkan teknik data mining (regresi, pohon keputusan, neural network, dll.) untuk hasilkan model.

A — Assess

Nilai performa model menggunakan metrik (akurasi, ROC) pada test set yang tidak pernah dilihat model.

🗄️ KDD — Knowledge Discovery in Databases (5 Tahap)
+
  1. Selection: Pilih data relevan dari database berdasarkan domain knowledge dan tujuan analisis.
  2. Preprocessing: Bersihkan data — hapus noise, tangani missing values, resolusi konflik antar sumber data.
  3. Transformation: Ubah format — normalisasi, agregasi, konstruksi fitur baru yang lebih representatif.
  4. Data Mining: Terapkan algoritma untuk menemukan pola tersembunyi (inti dari seluruh proses KDD).
  5. Interpretation/Evaluation: Interpretasikan pola, evaluasi relevansinya, visualisasikan hasilnya.
Perbandingan: KDD = paling akademik. CRISP-DM = paling banyak digunakan industri (tambah Business Understanding & Deployment). SEMMA = dikembangkan SAS untuk tool mereka.

5.3 Metode & Algoritma Data Mining

📈 PREDICTIVE TASKS

Menggunakan variabel yang diketahui untuk memprediksi variabel target. Ada variabel target (dependent variable).

  • Classification — target kategoris
  • Regression — target numerik kontinu
  • Time Series — prediksi berdasarkan waktu
🔍 DESCRIPTIVE TASKS

Mendeskripsikan pola dari data yang ada. TIDAK ada variabel target. Tidak memprediksi masa depan.

  • Clustering — kelompokkan data
  • Association Rule — hubungan antar item
  • Visualization — tampilkan pola visual
🌳 Decision Tree — Pohon Keputusan (Algoritma)
+
Apa: Membagi data secara rekursif menjadi cabang-cabang berdasarkan kondisi atribut, membentuk struktur pohon. Setiap leaf node = kelas/prediksi.
Algoritma populer: ID3C4.5/C5.0CARTCHAIDM5
Contoh use: Klasifikasi kredit (layak/tidak), diagnosis penyakit, klasifikasi spam.

4 Langkah Membangun Decision Tree:

  1. Buat root node dan assign semua data training ke dalamnya.
  2. Pilih atribut pemisah (splitting attribute) terbaik berdasarkan kriteria (Information Gain, Gini Index).
  3. Tambah cabang untuk setiap nilai split, bagi data ke subset mutually exclusive.
  4. Ulangi 2-3 untuk setiap leaf node hingga stopping criteria terpenuhi.
Splitting Criteria

Pilih atribut pemisah terbaik. Tujuan: ciptakan subset paling murni/homogen. Contoh: Information Gain (ID3), Gini Impurity (CART), Chi-Square (CHAID).

Stopping Criteria

Kapan berhenti: (1) Semua data di node sudah satu kelas, (2) Tidak ada atribut lagi, (3) Pohon mencapai kedalaman max, (4) Node terlalu kecil.

Pruning

Generalisasi pohon untuk mencegah overfitting. Pre-pruning: hentikan sebelum selesai. Post-pruning: bangun penuh lalu pangkas cabang tidak signifikan.

⚠️ KUIS PENTING: "Decision Tree" punya DUA arti! (1) Algoritma klasifikasi di Ch.5, (2) Representasi grafis scenario analysis di Ch.8. JANGAN tertukar!
🧠 Artificial Neural Network (ANN)
+
Apa: Model komputasi terinspirasi dari neuron otak manusia. Terdiri dari: input layer (menerima data), satu/lebih hidden layer (proses informasi), output layer (prediksi).
Cara kerja: Setiap koneksi punya bobot (weight). Selama training, bobot disesuaikan via backpropagation untuk meminimalkan error.
Kelebihan: Sangat akurat untuk data kompleks, non-linear.
Kekurangan: "Black Box" — sulit diinterpretasi. Butuh banyak data dan waktu lama.
Untuk clustering: Adaptive Resonance Theory (ART) dan Self-Organizing Map (SOM) adalah varian ANN untuk clustering.
Contoh use: Prediksi box-office film, pengenalan wajah, deteksi fraud.
⚔️ Support Vector Machine (SVM)
+
Apa: Menemukan hyperplane optimal yang memisahkan kelas dengan margin terlebar. Data paling dekat dengan hyperplane = support vectors.
Intuisi: Memisahkan apel dan jeruk dengan garis (2D) / bidang (3D) dengan jarak terbesar antara apel terdekat dan jeruk terdekat.
Kernel Trick: Untuk data non-linearly separable, gunakan kernel function untuk memetakan ke dimensi lebih tinggi.
Performa (film): Bingo: 55.49% — Terbaik di individual models
Kelebihan: Efektif di high-dimensional spaces, robust terhadap overfitting.
Contoh use: Klasifikasi teks, bioinformatik, prediksi sukses film.
🔵 k-Means Clustering — 4 Langkah Wajib Hafal
+
Apa: Algoritma unsupervised yang membagi data ke k cluster berdasarkan kemiripan. Tidak ada variabel target — menemukan struktur alami data.
  1. Tentukan k (jumlah cluster) dan pilih k centroid awal secara acak.
  2. Assign setiap titik ke centroid terdekat (berdasarkan jarak Euclidean).
  3. Hitung ulang centroid baru berdasarkan rata-rata semua titik dalam setiap cluster.
  4. Ulangi langkah 2-3 sampai convergence — assignment titik ke cluster tidak berubah lagi.
Tantangan utama: Menentukan nilai k optimal. Metode: Elbow Method, Silhouette Score. Sensitif terhadap outlier dan pemilihan centroid awal.
Algoritma lain: Fuzzy c-means, ART, SOM, Genetic Algorithms.
🛒 Association Rule Mining — Market Basket Analysis
+
Apa: Menemukan aturan yang menyatakan item mana yang cenderung muncul bersama dalam transaksi.
Format: {A, B} → {C}: "Orang yang beli A dan B cenderung juga beli C"
Contoh klasik: {Roti, Susu} → {Mentega}. Juga: {Popok} → {Bir}
Metriks: Support Confidence Lift
Algoritma utama: Apriori Algorithm
Strategi: Berdekatan

Letakkan item yang sering dibeli bersama di rak yang berdekatan → mudah dijangkau, tingkatkan penjualan keduanya.

Strategi: Bundle

Promosikan sebagai paket/bundling. Contoh: "Beli roti + susu, gratis mentega."

Strategi: Jauhkan

Letakkan item berjauhan → pelanggan telusuri lebih banyak area toko, meningkatkan exposure produk lain.

🌲 Ensemble Methods — Kekuatan Gabungan Model
+
Apa: Menggabungkan beberapa model untuk prediksi lebih akurat dan stabil. Seperti keputusan berdasarkan banyak ahli — kesalahan tiap model saling mengoreksi.
Random Forest

Banyak pohon keputusan acak (subset acak data+fitur), gabungkan dengan voting. Bingo: 54.62%, 1-Away: 89.60%, SD: 0.76

Boosted Tree

Pohon berurutan — setiap pohon fokus pada kesalahan pohon sebelumnya. Bingo: 54.05%, 1-Away: 84.10%, SD: 0.84

⭐ Fusion (Average)

Rata-rata prediksi semua model. TERBAIK KESELURUHAN: Bingo: 56.07%, 1-Away: 90.75%, SD: 0.63 (terendah!)

Keunggulan Ensemble: Standard deviation jauh lebih rendah → prediksi lebih konsisten. Fusion SD=0.63 vs CART SD=1.05.

5.4 Evaluasi Model & Studi Kasus Hollywood

📏 5 Kriteria Evaluasi + k-Fold CV + ROC Curve
+
1
Predictive Accuracy
Seberapa sering model membuat prediksi benar pada holdout sample. Diukur dengan % hit rate. Contoh: SVM Bingo=55.49%, Fusion=56.07%
2
Speed
Dua dimensi: (a) kecepatan training, (b) kecepatan scoring/prediksi. Penting untuk aplikasi real-time.
3
Robustness
Performa tetap baik meskipun ada data noisy, missing values, atau distribusi sedikit berbeda dari training data.
4
Scalability
Kemampuan mempertahankan efisiensi saat data meningkat drastis (dari ribuan ke jutaan record).
5
Interpretability / Transparency
Seberapa mudah model dapat dipahami manusia. DT = sangat interpretable. ANN = black box. Ada tradeoff: akurasi vs interpretability.
k-Fold Cross Validation

Bagi data ke k bagian. Model dilatih k kali — setiap kali 1 fold jadi test set. Hasil = rata-rata k percobaan. Lebih andal dari split biasa karena setiap data point digunakan sebagai test set tepat sekali.

ROC Curve & AUC

Grafik True Positive Rate vs False Positive Rate. AUC = Area Under ROC Curve.
• AUC=0.5 → tidak lebih baik dari tebakan acak
• AUC=1.0 → klasifikasi sempurna
• AUC=0.84 → contoh dari materi (sangat baik)
Cocok untuk skewed class distributions.

8.1 Prescriptive Analytics & Linear Programming

Prescriptive Analytics: Menjawab "Apa yang sebaiknya dilakukan?" — level analitik tertinggi (Descriptive → Predictive → Prescriptive). Memberikan rekomendasi tindakan optimal.
Linear Programming (LP): Model optimisasi terbaik yang dikenal. Alokasi optimal sumber daya terbatas di antara aktivitas bersaing. SEMUA hubungan matematis harus linear.
📐 4 Komponen Model LP + Slack/Surplus
+
🎯
1. Decision Variables
Variabel yang DIKONTROL pengambil keputusan. Nilainya yang dicari algoritma optimisasi. Mewakili "apa yang bisa kita atur."
Contoh MBI: X₁=unit CC-7 yang diproduksi; X₂=unit CC-8 yang diproduksi
📈
2. Objective Function
Fungsi yang ingin dioptimalkan — dimaksimalkan (profit) atau diminimalkan (biaya). Ini adalah "tujuan" model.
Contoh MBI: Max Z = 8.000X₁ + 12.000X₂
🔒
3. Constraints
Batasan yang merepresentasikan teknologi, kondisi pasar, variabel tidak terkontrol. Membatasi ruang solusi yang valid.
Contoh MBI: 300X₁+500X₂≤200.000 (labor); X₁≥100 (min CC-7)
📊
4. Result Variable (Z)
Nilai output dari model berdasarkan decision variables yang dipilih. Ini adalah "jawaban" model.
Contoh MBI: Z = Total Profit = $5.066.667
Slack Variable (komponen tersembunyi)

Muncul pada ≤ constraints. Merepresentasikan kapasitas/budget yang tidak terpakai (kelebihan sumber daya). Contoh: labor yang tersisa setelah produksi.

Surplus Variable (komponen tersembunyi)

Muncul pada ≥ constraints. Merepresentasikan produksi yang melebihi minimum yang diminta. Contoh: unit lebih dari minimum marketing requirement.

8.2 Sensitivity Analysis, What-If & Goal Seeking

📉 Sensitivity Analysis — Definisi, Penggunaan & 2 Jenis
+
Sensitivity Analysis: Menilai dampak perubahan pada data input atau parameter terhadap solusi yang diusulkan. "Seberapa robustkah solusi optimal kita jika asumsi berubah?"

Mengapa penting:

  • Fleksibilitas dan adaptasi terhadap kondisi yang berubah
  • Pemahaman yang lebih baik tentang model dan situasi keputusan
  • Meningkatkan kepercayaan manajer terhadap model
Automatic Sensitivity Analysis

Dilakukan otomatis oleh LP software. Melaporkan range nilai parameter yang bisa berubah tanpa mempengaruhi solusi signifikan.

Keterbatasan: Hanya satu perubahan sekaligus, hanya variabel tertentu.

Disediakan oleh Solver, Lindo, hampir semua software LP.

Trial-and-Error Sensitivity Analysis

Ubah input manual, solve ulang, bandingkan. Bisa banyak perubahan sekaligus. Dua pendekatan:

• What-If Analysis — ubah input, lihat output
• Goal Seeking — tentukan output, cari input

↔️ What-If Analysis vs Goal Seeking — Perbedaan KRITIS
+
🔮 What-If Analysis (FORWARD)

"Jika saya ubah INPUT ini, apa yang terjadi pada OUTPUT?"

Contoh nyata:
Initial sales 100→120, growth 3%→4%
Hasil: profit naik $127 → $182

Contoh lain:
• Apa yang terjadi pada inventory cost jika biaya penyimpanan naik 10%?
• Apa yang terjadi pada market share jika advertising naik 5%?

🎯 Goal Seeking (BACKWARD)

"Nilai INPUT apa yang dibutuhkan untuk mencapai OUTPUT tertentu?"

Contoh nyata:
Set NPV=0, cari interest rate
Jawaban: IRR = 38.77059%

Contoh lain:
• Anggaran R&D berapa untuk pertumbuhan 15%/tahun?
• Berapa perawat untuk kurangi waktu tunggu <10 menit?

Break-Even Point via Goal Seeking: Menentukan nilai decision variable yang menghasilkan profit=0. "Pada volume berapa kami mulai untung?"
🎯 Multiple Goals — Definisi, 7 Kesulitan, 4 Metode
+

Kebanyakan masalah manajerial memiliki BANYAK tujuan yang bisa saling konflik. Contoh: perusahaan ingin sekaligus maksimalkan profit, tumbuhkan bisnis, kembangkan karyawan, layani komunitas.

7 Kesulitan Multiple Goals:

  1. Sulit mendapatkan pernyataan eksplisit tujuan organisasi
  2. Pengambil keputusan dapat mengubah prioritas tujuan seiring waktu
  3. Tujuan dipandang berbeda di berbagai level dan departemen
  4. Tujuan berubah seiring perubahan lingkungan
  5. Hubungan antara alternatif dan tujuan sulit dikuantifikasi
  6. Masalah kompleks diselesaikan kelompok dengan agenda berbeda
  7. Peserta menilai kepentingan tujuan secara berbeda

8.3 Decision Analysis & Simulasi

💰 Decision Table, Decision Tree & Contoh Investasi Lengkap
+
Decision Table

Mengorganisir informasi secara tabular. 3 komponen: (1) Decision variables (alternatives), (2) Uncontrollable variables (states of nature), (3) Result variables (outcomes).

Decision Tree (Skenario)

Representasi GRAFIS dari decision table. Lebih intuitif untuk situasi kompleks. Tools: TreeAge Pro, PrecisionTree (Palisade).
BUKAN algoritma klasifikasi!

Contoh Investasi (Risk vs Uncertainty):

AlternatifSolid Growth (50%)Stagnation (30%)Inflation (20%)Expected Value
Bonds12%6%3%8.4%
Stocks15%3%-2%8.0%
CDs6.5%6.5%6.5%6.5%
EV Bonds = 12(0.5) + 6(0.3) + 3(0.2) = 6 + 1.8 + 0.6 = 8.4%
Treating UNCERTAINTY (tanpa prob)
  • Optimistic (Maximax): Best of best → Stocks (15%)
  • Pessimistic (Maximin): Best of worst → CDs (6.5%)
  • Equal Probability: Semua states sama mungkin
Treating RISK (dengan prob)

Gunakan Expected Value = Σ (outcome × probability).

Pilih alternatif EV tertinggi → Bonds (8.4%)

⚠️ Bahaya EV: Bisa menyesatkan jika ada risiko katastrofik kecil. Contoh: EV=$949.80 tapi 0.01% rugi $500.000 → tidak semua investor bisa menanggung!
⚙️ Simulasi — Definisi, Jenis, Proses Lengkap
+
Simulasi: Teknik untuk melakukan eksperimen dengan komputer pada model sistem manajemen. Bersifat DESKRIPTIF, bukan preskriptif — tidak ada pencarian solusi optimal otomatis. Digunakan HANYA ketika optimisasi tidak memungkinkan.

4 Karakteristik Simulasi:

  1. Membangun model realita sedekat mungkin yang praktis
  2. Melibatkan lebih sedikit asumsi dibanding model optimisasi lainnya
  3. Teknik untuk melakukan eksperimen — mengetes nilai variabel tertentu
  4. Digunakan HANYA ketika masalah terlalu kompleks untuk optimisasi numerik
Deterministic vs Stochastic

Deterministic: Tidak ada randomness, output pasti.
Stochastic: Menggunakan distribusi probabilitas untuk ketidakpastian.

Monte Carlo

Teknik dengan bilangan acak. Formula: X = A + B di mana A, B, X adalah distribusi (bukan nilai tetap). Berguna untuk risk analysis keuangan.

Discrete Event vs Continuous

Discrete: Perubahan pada titik waktu tertentu (antrian bank).
Continuous: Perubahan terus-menerus (aliran cairan).

Implementasi: Visual Simulation Object-Oriented Agent-based
Software: Arena ExtendSim SAS Simulation Studio Simio

6 Langkah Proses Simulasi:

  1. Define problem — definisikan masalah dan tujuan
  2. Construct the model — bangun model matematis/logis
  3. Test and validate model — pastikan model merepresentasikan realita
  4. Design/conduct experiments — rancang dan jalankan skenario
  5. Evaluate results — analisis output, bandingkan skenario
  6. Implement solution — terapkan temuan ke dunia nyata

11.3 Responsible & Ethical AI — 6 Principles Lengkap

🛡️ 6 Principles of Responsible AI — Definisi + Contoh + Kegagalan
+
⚖️
1. FAIRNESS (Keadilan)
Definisi: AI harus memperlakukan semua orang secara adil dan menghindari bias/diskriminasi berdasarkan ras, jenis kelamin, usia, agama, atau karakteristik lainnya.
Mengapa penting: AI belajar dari data historis. Jika data historis biased → model akan mereproduksi bias dalam skala besar.
Kegagalan — Amazon AI Recruiting: Alat rekrutmen AI mendiskriminasi pelamar perempuan karena dilatih pada data rekrutmen historis yang didominasi pria. Akhirnya dihapus.
Kegagalan — COMPAS: Sistem prediksi residivisme dituduh bias rasial terhadap orang kulit hitam.
🔧
2. RELIABILITY & SAFETY (Keandalan & Keselamatan)
Definisi: AI harus berperforma secara andal dan aman dalam berbagai kondisi — termasuk kondisi tidak terduga. Tidak boleh gagal dengan cara yang membahayakan manusia.
Mengapa penting: AI digunakan dalam sistem kritis (kendaraan otonom, diagnosis medis) di mana kegagalan bisa fatal.
Contoh penerapan: Kendaraan otonom harus reliabel dalam cuaca buruk, malam hari, dan situasi tidak biasa — bukan hanya kondisi ideal.
Risiko: Robot di lini produksi yang gagal mendeteksi kehadiran manusia → kecelakaan serius.
🔒
3. PRIVACY & SECURITY (Privasi & Keamanan)
Definisi: AI harus menghormati privasi pengguna dan melindungi data dari akses tidak sah. Pengumpulan dan penggunaan data harus transparan dan dengan persetujuan pengguna.
Mengapa penting: AI membutuhkan data besar, seringkali sensitif (kesehatan, keuangan, lokasi). Pelanggaran bisa sangat merugikan.
Contoh penerapan: Platform yang pakai AI untuk personalisasi konten harus jelaskan data apa yang dikumpulkan dan berikan opsi opt-out.
Kegagalan nyata: Platform media sosial yang gunakan data pengguna untuk melatih model tanpa persetujuan eksplisit.
🌍
4. INCLUSIVENESS (Inklusivitas)
Definisi: AI harus memberdayakan SEMUA orang dan melibatkan semua pemangku kepentingan. Harus dapat diakses dan bermanfaat bagi semua segmen masyarakat, bukan hanya kelompok tertentu.
Mengapa penting: Jika dikembangkan hanya untuk kelompok demografis tertentu → gagal melayani populasi lebih luas secara efektif.
Contoh penerapan: Asisten suara yang bekerja baik untuk berbagai aksen dan bahasa, bukan hanya aksen Inggris Amerika standar.
Kegagalan nyata: Sistem pengenalan wajah akurat untuk wajah putih tapi kurang akurat untuk wajah kulit gelap karena training data tidak representatif.
🔭
5. TRANSPARENCY (Transparansi)
Definisi: AI harus dapat dipahami dan dijelaskan (explainable/interpretable). Pengambil keputusan harus bisa mengetahui MENGAPA AI membuat keputusan tertentu.
Mengapa penting: Kepercayaan pada AI bergantung pada pemahaman cara kerjanya. "Black box" AI sulit diaudit, dipertanggungjawabkan, atau diperbaiki.
Catatan: Ada tradeoff — model lebih kompleks = lebih akurat tapi kurang transparan (ANN).
Contoh penerapan: Model kredit bisa menjelaskan: "Pinjaman ditolak karena rasio utang/pendapatan 45% (di atas ambang 40%) dan 3 kali terlambat bayar."
Persyaratan model AI: Jika model tidak bisa dipahami, penerapannya bisa menyebabkan konsekuensi tidak terduga.
📋
6. ACCOUNTABILITY (Akuntabilitas)
Definisi: Harus ada pihak yang bertanggung jawab atas keputusan AI. Model harus memiliki mekanisme self-correcting dan diaudit secara reguler. Input dan output harus terus dievaluasi.
Mengapa penting: Ketika AI membuat keputusan yang salah dan merugikan, harus ada pertanggungjawaban yang jelas dan mekanisme perbaikan.
Contoh penerapan: Perusahaan yang gunakan AI untuk kredit harus punya tim audit reguler, proses appeal yang jelas, dan mekanisme koreksi keputusan salah.
Pertanyaan kunci: Jika kendaraan otonom menabrak pejalan kaki, siapa yang bertanggung jawab — pengemudi, produsen, atau pembuat software AI?
📜 10 Commandments of Computer Ethics — Penjelasan Lengkap
+
1
Jangan gunakan komputer untuk menyakiti orang lain
Termasuk: cyberbullying, doxxing, membuat malware, menggunakan data untuk memanipulasi dan merugikan orang.
2
Jangan ganggu pekerjaan komputer orang lain
Termasuk: mengirim spam, DDoS attack, merusak jaringan atau infrastruktur komputer orang lain.
3
Jangan mengintip file orang lain
Mengakses email, dokumen, atau data privat tanpa izin — meskipun secara teknis memungkinkan — adalah pelanggaran privasi.
4
Jangan gunakan komputer untuk mencuri
Termasuk: hacking ke rekening bank, pencurian identitas digital, phishing, mengakses sistem keuangan ilegal.
5
Jangan gunakan komputer untuk bersaksi palsu
Menyebarkan disinformasi, membuat deepfake, memanipulasi data digital untuk penipuan atau konteks hukum.
6
Jangan gunakan/salin software tanpa membayar
Pembajakan software melanggar hak cipta. Termasuk: software crack, key generator, distribusi ilegal software berlisensi.
7
Jangan gunakan sumber daya komputer orang lain tanpa izin
Termasuk menggunakan server, bandwidth, atau kapasitas orang lain tanpa izin (contoh: cryptojacking).
8
Jangan ambil alih output intelektual orang lain
Plagiarisme digital, mengklaim karya orang lain sebagai milik sendiri, mencuri kode sumber tanpa atribusi.
9
Pikirkan konsekuensi sosial dari program yang ditulis
Developer bertanggung jawab atas dampak sosial software mereka. Algoritma biased atau tools surveillance harus dipertimbangkan dampaknya.
10
Selalu gunakan komputer dengan pertimbangan dan respek
Prinsip umum: pertimbangkan dampak penggunaan teknologi terhadap orang lain. Teknologi adalah alat — penggunaannya harus mencerminkan nilai etis.
🤖 3 Persyaratan Model AI yang Bertanggung Jawab
+
I
Transparan (Explainable)
Model harus dapat dipahami dan dijelaskan. Jika model tidak bisa dipahami → penerapannya bisa menyebabkan konsekuensi tidak terduga yang sulit diperbaiki. Kadang ada tradeoff: model lebih kompleks = lebih akurat tapi kurang transparan.
II
Tujuan Kuantitatif yang Jelas
Model harus memiliki tujuan yang TERUKUR secara objektif. Tanpa tujuan jelas → tidak mungkin mengevaluasi keberhasilan atau standar perbaikan. Contoh BAIK: "Kurangi false positive rate di bawah 5%." Contoh BURUK: "Buat model lebih baik."
III
Mekanisme Self-Correcting + Audit Reguler
Harus ada proses untuk mendeteksi dan mengoreksi kesalahan. Diaudit secara reguler dengan input dan output yang terus dipertimbangkan. Alasan: dunia berubah → concept drift → model yang akurat hari ini mungkin tidak akurat besok.
